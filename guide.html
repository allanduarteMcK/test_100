What is GitHub?

A short, visual guide to the platform that powers how millions of developers build and ship software together.

Git vs GitHub

Git is a version-control system created by Linus Torvalds. It runs on your computer and tracks changes in your project over time: who changed what, when, and why. You can rewind, branch, and merge history without losing work.

GitHub is a company and a website built on top of Git. It hosts your Git repositories in the cloud, adds a web interface, and adds collaboration features: pull requests, issues, code review, Actions, and more. Think of Git as the engine; GitHub is the place where people use that engine together.

Git is the tool. GitHub is where the world builds with it.

Key concepts

These ideas show up everywhere once you start using GitHub.

Repository

A project folder that Git tracks. It contains your files and the full history of changes. Can be local (on your machine) or remote (on GitHub).

Commit

A saved snapshot of your project at a point in time. Each commit has a message, an author, and a unique ID. History is a chain of commits.

Branch

A parallel line of work. You branch off from main to try an idea, then merge back when it’s ready. Keeps main stable.

Pull request

A request to merge your branch into another (often main). Lets others review your code and discuss before anything is merged.

Fork

Your own copy of someone else’s repo on GitHub. You can change your fork and then open a pull request back to the original project.

Clone

Downloading a repository from GitHub (or another remote) to your computer so you have a full local copy and can work offline.

A simple workflow

This is the loop most teams use every day.

  1. Clone the repo (or pull the latest changes if you already have it).
  2. Create a branch for your feature or fix (e.g. feature/login-form).
  3. Make changes and commit them with clear messages.
  4. Push your branch to GitHub.
  5. Open a pull request into main and get review.
  6. After approval, merge the PR. Your changes are now in main.

Why it matters

GitHub is where open source lives: Linux, React, VS Code, and countless libraries are developed there. It’s also where many companies keep their private code and run their CI/CD. Learning Git and GitHub is learning how modern software is built and shipped.

Quick glossary

main / master
The default branch name in a repo. Often where “production-ready” code lives.
merge
Combining two branches so that one branch’s commits become part of another.
push / pull
Push = send your commits to a remote (e.g. GitHub). Pull = fetch and merge changes from the remote into your current branch.
staging
Marking changes to be included in the next commit (e.g. git add).
remote
A version of the repo hosted elsewhere (e.g. origin on GitHub).
